Return a float representing the current system-wide CPU
utilization as a percentage.

When *interval* is > 0.0 compares system CPU times elapsed before
and after the interval (blocking).

When *interval* is 0.0 or None compares system CPU times elapsed
since last call or module import, returning immediately (non
blocking). That means the first time this is called it will
return a meaningless 0.0 value which you should ignore.
In this case is recommended for accuracy that this function be
called with at least 0.1 seconds between calls.

When *percpu* is True returns a list of floats representing the
utilization as a percentage for each CPU.
First element of the list refers to first CPU, second element
to second CPU and so on.
The order of the list is consistent across calls.

Examples:

  >>> # blocking, system-wide
  >>> psutil.cpu_percent(interval=1)
  2.0
  >>>
  >>> # blocking, per-cpu
  >>> psutil.cpu_percent(interval=1, percpu=True)
  [2.0, 1.0]
  >>>
  >>> # non-blocking (percentage since last call)
  >>> psutil.cpu_percent(interval=None)
  2.9
  >>>

Return system disk I/O statistics as a namedtuple including
the following fields:

 - read_count:  number of reads
 - write_count: number of writes
 - read_bytes:  number of bytes read
 - write_bytes: number of bytes written
 - read_time:   time spent reading from disk (in ms)
 - write_time:  time spent writing to disk (in ms)

Platform specific:

 - busy_time: (Linux, FreeBSD) time spent doing actual I/Os (in ms)
 - read_merged_count (Linux): number of merged reads
 - write_merged_count (Linux): number of merged writes

If *perdisk* is True return the same information for every
physical disk installed on the system as a dictionary
with partition names as the keys and the namedtuple
described above as the values.

If *nowrap* is True it detects and adjust the numbers which overflow
and wrap (restart from 0) and add "old value" to "new value" so that
the returned numbers will always be increasing or remain the same,
but never decrease.
"disk_io_counters.cache_clear()" can be used to invalidate the
cache.

On recent Windows versions 'diskperf -y' command may need to be
executed first otherwise this function won't find any disk.

 - ptp: stands for "point to point" and references the

 - broadcast: and *ptp* are mutually exclusive.

Note: you can have more than one address of the same family
associated with each interface.

def pylo.vendors.psutil.wait_procs (   procs,

Convenience function which waits for a list of processes to
terminate.

Return a (gone, alive) tuple indicating which processes
are gone and which ones are still alive.

The gone ones will have a new *returncode* attribute indicating
process exit status (may be None).

*callback* is a function which gets called every time a process
terminates (a Process instance is passed as callback argument).

Function will return as soon as all processes terminate or when
*timeout* occurs.
Differently from Process.wait() it will not raise TimeoutExpired if
*timeout* occurs.

Typical use case is:

 - send SIGTERM to a list of processes
 - give them some time to terminate
 - send SIGKILL to those ones which are still alive

Example:

>>> def on_terminate(proc):
...     print("process {} terminated".format(proc))
...
>>> for p in procs:
...    p.terminate()
...
>>> gone, alive = wait_procs(procs, timeout=3, callback=on_terminate)
>>> for p in alive:

1594  """Convenience function which waits for a list of processes to
1595  terminate.
1596 
1597  Return a (gone, alive) tuple indicating which processes
1598  are gone and which ones are still alive.
1599 
1600  The gone ones will have a new *returncode* attribute indicating
1601  process exit status (may be None).
1602 
1603  *callback* is a function which gets called every time a process
1604  terminates (a Process instance is passed as callback argument).
1605 
1606  Function will return as soon as all processes terminate or when
1607  *timeout* occurs.
1608  Differently from Process.wait() it will not raise TimeoutExpired if
1609  *timeout* occurs.
1610 
1611  Typical use case is:
1612 
1613  - send SIGTERM to a list of processes
1614  - give them some time to terminate
1615  - send SIGKILL to those ones which are still alive
1616 
1617  Example:
1618 
1619  >>> def on_terminate(proc):
1620  ... print("process {} terminated".format(proc))
1621  ...
1622  >>> for p in procs:
1623  ... p.terminate()
1624  ...
1625  >>> gone, alive = wait_procs(procs, timeout=3, callback=on_terminate)
1626  >>> for p in alive:
1627  ... p.kill()
1628  """
1629  def check_gone(proc, timeout):
1630  try:
1631  returncode = proc.wait(timeout=timeout)
1632  except TimeoutExpired:
1633  pass
1634  else:
1635  if returncode is not None or not proc.is_running():
1636  proc.returncode = returncode
1637  gone.add(proc)
1638  if callback is not None:
1639  callback(proc)
1640 
1641  if timeout is not None and not timeout >= 0:
1642  msg = "timeout must be a positive integer, got %s" % timeout
1643  raise ValueError(msg)
1644  gone = set()
1645  alive = set(procs)
1646  if callback is not None and not callable(callback):
1647  raise TypeError("callback %r is not a callable" % callable)
1648  if timeout is not None:
1649  deadline = _timer() + timeout
1650 
1651  while alive:
1652  if timeout is not None and timeout <= 0:
1653  break
1654  for proc in alive:
1655  # Make sure that every complete iteration (all processes)
1656  # will last max 1 sec.
1657  # We do this because we don't want to wait too long on a
1658  # single process: in case it terminates too late other
1659  # processes may disappear in the meantime and their PID
1660  # reused.
1661  max_timeout = 1.0 / len(alive)
1662  if timeout is not None:

1664  if timeout <= 0:
1665  break
1666  check_gone(proc, timeout)
1667  else:
1668  check_gone(proc, max_timeout)
1669  alive = alive - gone
1670

1673  # timeout == 0 won't make this function wait any further.
1674  for proc in alive:
1675  check_gone(proc, 0)
1676  alive = alive - gone
